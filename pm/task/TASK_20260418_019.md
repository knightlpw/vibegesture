# TASK_20260418_019

## 本轮目标
本轮只完成：收敛菜单栏与设置页中的调试/诊断信息，让一般用户默认只看到简洁、易理解、能操作的界面；本轮不改底层识别、状态机、键盘、gate、权限或摄像头逻辑。


## 本轮背景

- 当前应用已经可以稳定运行并对外安装，但菜单栏与设置面板里仍然暴露了大量内部诊断信息
- 这些信息对开发和验收有用，但对一般用户来说过于技术化，会影响理解成本和产品观感
- 用户明确要求：只做用户友好页面优化，不要改掉原来的逻辑

## 本轮要做

- 梳理菜单栏与设置页中哪些信息应归为“普通用户可见”
- 将内部诊断字段统一收进诊断区域，默认折叠或隐藏
- 让普通用户界面尽量只保留“当前能不能用、为什么不能用、下一步怎么做”这类信息

## 本轮不要做

- 不要改 `GestureInterpreter`、`RecognitionStateMachine`、`KeyboardDispatcher` 的运行逻辑
- 不要改权限判定、foreground gate、摄像头、录音、submit/cancel 语义
- 不要把调试信息从代码里彻底删除，先只做界面收敛
- 不要引入新的识别模式、训练模式或校准流程

## 本轮主流程
1. 先定义普通用户默认可见的信息集合
2. 再把菜单栏和设置页里的诊断字段收进可折叠 / 可隐藏区域
3. 保留必要的排障能力，但默认不打扰一般用户
4. 最后验证界面看起来更简洁，同时原有逻辑不受影响

## 建议实现内容

以下不是唯一实现方式，但请保持目标一致：

- 菜单栏默认只保留最少量状态摘要
- 设置页默认只保留用户会实际操作的权限、快捷键、校准入口
- 诸如 `Gesture candidate`、`Gesture pose`、`Recent action`、`Keyboard result`、`Pipeline state`、`Latest observation`、`Calibration source`、`Saved samples`、`Working samples` 之类字段收进诊断区
- 诊断区可以默认折叠，或者通过显式开关打开
- 如果需要保留开发者可见入口，优先做成“用户看不见、开发者能展开”的轻量方案

## 验收标准

- [ ] 默认界面比当前更简洁，一般用户不会一眼看到大量内部诊断字段
- [ ] 菜单栏与设置页的诊断信息已被收敛到隐藏 / 折叠区域
- [ ] 原有识别、状态机、键盘、gate、权限逻辑未被修改
- [ ] `swift build` 和 `swift test` 通过
