本轮只完成：搭建默认摄像头采集与 Vision hand pose pipeline 骨架。

- TASK001 已完成并验证通过
- TASK002 已完成并验证通过
- 当前仓库已经有可编译的 macOS 菜单栏 shell、配置骨架、权限状态模型、权限检查器、受限状态展示与轻量系统设置引导

## 本轮要做

- 新增默认摄像头采集管理器
- 把摄像头帧接入 Vision hand pose detection
- 建立一条稳定的 frame -> observation 数据流
- 让采集状态可以被现有 AppCoordinator 启停
- 在设置或状态展示里反映摄像头 / pipeline 的基础状态
- 保留后续手势解释所需的观测接口，但不做手势分类

## 本轮不要做

- 不要写 pinch / submit / cancel 判定
- 不要写完整 state machine 细节
- 不要写前台应用 gating
- 不要写键盘事件发射
- 不要把设置界面扩展成完整偏好设置产品
- 不要提前接入 Typeless 或外部应用内部状态

## 本轮主流程
1. 新增默认摄像头采集组件
2. 把摄像头帧送入 Vision hand pose 请求
3. 输出标准化的 hand pose observation 或 frame observation 结果
4. 让识别启用 / 禁用可以控制采集开始和停止
5. 在轻量 UI 中显示采集 / pipeline 的基础状态

---

## 建议实现内容

以下不是唯一实现方式，但请保持目标一致：

- Swift + 现有 AppCoordinator / AppState 结构继续扩展
- 用独立的 camera manager 封装 `AVCaptureSession`
- 用独立的 Vision processor 封装 hand pose 请求
- 只使用默认摄像头
- 只处理右手观测
- 用一个清晰的数据模型承载单帧观测结果，供后续手势解释阶段使用
- 当识别关闭时，摄像头采集应停止
