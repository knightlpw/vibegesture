# TASK_20260414_002.md

## 本轮目标
本轮只完成：搭建权限管理与识别受限状态骨架。

---

## 本轮背景

- TASK001 已完成并验证通过
- 当前仓库已经有可编译的 macOS 菜单栏 shell、配置骨架、全局快捷键入口和轻量设置入口
- 当前实现仍然没有摄像头、Vision、手势解释、app gating 或键盘发射
- roadmap 的第 2 阶段是权限与安全关停
- 下一步应先把 Camera / Accessibility 权限状态和识别受限状态接稳，再往后接摄像头和手势逻辑

---

## 本轮要做

- 新增权限状态模型和权限检查器
- 在现有 AppCoordinator / AppState 里接入权限状态
- 启动时检查 Camera 和 Accessibility 权限
- 当权限缺失时，给出清晰的受限状态和轻量引导入口
- 让识别开关在权限缺失时无法进入正常识别态
- 预留一个安全关停接口，供后续录音 / 键盘发射阶段使用

---

## 本轮不要做

- 不要接摄像头采集
- 不要接 Vision hand pose detection
- 不要写手势解释逻辑
- 不要写完整 state machine 细节
- 不要写前台应用 gating
- 不要写键盘事件发射
- 不要把设置界面扩展成完整偏好设置产品

---

## 本轮主流程
1. 新增 `PermissionState` / `PermissionManager` 之类的权限模型与检查入口
2. 在应用启动时读取并刷新权限状态
3. 将权限状态接入现有识别开关流程，让缺权限时进入受限状态
4. 让菜单栏和设置入口能看见当前权限状态
5. 预留安全关停 hook，为后续录音停止动作留接口

---

## 建议实现内容

以下不是唯一实现方式，但请保持目标一致：

- Swift + 现有 AppCoordinator / AppState 结构继续扩展
- 用单独的权限管理对象封装 Camera / Accessibility 检查
- 用一个明确的 `PermissionState` 表达 `ready` / `missing_camera` / `missing_accessibility` / `missing_both`
- 识别开关在权限未就绪时应进入 `errorPermissionMissing`，而不是进入正常识别态
- 轻量设置界面可以展示缺失权限说明和引导文案，但不要做成完整 onboarding

---
